## Why this matters

The R3 audit caught real bugs (Stage 1 false-green, D-32 runtime explosion). The forensics asked a sharper question: *why did 5 commits in the previous session ship with formatting drift and an F541 that CI caught?* Answer: the contributor clone had ZERO hooks installed in `.git/hooks/`. The first filter was opt-in and silently absent. v1.12.0 makes the first filter the contract: `make install-dev` now verifies hooks landed; `make verify-hooks` audits any time; both pre-commit and pre-push are installed by default; CI runs the same hooks as the safety net, not the gate.

## Upgrade path

```bash
git pull
make install-dev    # idempotent: installs new mypy/bandit hooks + verifies setup
make verify-hooks   # audit hooks landed
```

If `verify-hooks` fails, `bash scripts/dev-setup.sh` is the canonical fix.
